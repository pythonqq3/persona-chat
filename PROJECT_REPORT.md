# 🏆 张仕达数字分身 · 项目总结报告

> 始于 2026 年 5 月 30 日，一个 19 岁大学生的微信聊天记录，变成了一座能说话的数字分身。

---

## 一、项目概述

### 做了什么

把 2 年微信聊天数据（6 个对话窗口、24,000 条消息）"蒸馏"成 AI 人格，部署为公网可访问的聊天网站。任何人打开浏览器就能和"张仕达"对话——语气、思维、知识，都像本人。

### 最终成果

| 产物 | 地址 |
|------|------|
| **公开网页** | `https://chat.061230zsd.xyz` |
| **直连地址** | `https://zhangshida.streamlit.app` |
| **源代码** | `https://github.com/pythonqq3/persona-chat` |
| **人格模型文件** | 本地保存（15GB merged LoRA） |

## 二、技术架构

```
原始数据(WeFlow TXT)
    ↓ 清洗脚本
24,060 条纯文本消息
    ↓ 人格分析(高频词/句式/情感/话题)
SKILL.md (400+ 行人格规则)
    ↓ + TF-IDF 检索(21,000 条例句)
系统提示词 + 少样本示例
    ↓ DeepSeek API
对话回复
    ↓ Streamlit 网页 + GitHub Pages 域名
用户访问
```

### 技术栈

| 层 | 技术 |
|----|------|
| **人格引擎** | DeepSeek API（千亿参数大模型） |
| **检索增强** | TF-IDF + 语义相似度（24,000 条索引） |
| **网页框架** | Streamlit |
| **部署平台** | Streamlit Cloud（免费） |
| **域名** | 阿里云 `061230zsd.xyz` + GitHub Pages |
| **微调实验** | LLaMA-Factory + Qwen2.5-7B + LoRA（AutoDL A100） |

## 三、项目历程

### Phase 1：数据准备（5 月 30 日上午）

**做什么**：用 WeFlow 从微信 PC 端导出聊天记录，组织为 6 个文件夹。

**技术细节**：
- 编写 `clean_chat.py` 解析 WeFlow TXT 格式（时间戳 + 说话人 + 内容）
- 过滤系统消息、表情包、语音标记
- 提取所有"我"的发言——24,060 条，总共 235,951 字
- 输出 `cleaned_messages.json` + `cleaned_messages.txt`

**关键决策**：只保留自己的发言，不保留对方的。这是做数字分身的基础——你只需要自己的语言。

### Phase 2：人格分析（5 月 30 日中午）

**做什么**：对 24,000 条消息做多维度统计分析。

**发现的关键特征**：
- 平均消息长度 9.7 字，极短句风格
- 句号使用率 1.1%，几乎不打标点
- 最高频词：就是(685)、然后(538)、六六六(271)、奥奥(252)
- 最活跃时段：晚上 21-23 时
- Emoji 使用率：0.4%
- 核心爱好：羽毛球、写诗词、王者荣耀、跑步、道家

**关键决策**：不只是看"说什么"，还要分析"什么时候说""怎么回应""情绪怎么表达"。

### Phase 3：SKILL.md 人格文件（5 月 30 日下午）

**做什么**：把分析结果写作 LLM 可读的人格规则文件。

**迭代次数**：5 个大版本，无数次小改。

| 版本 | 核心改进 |
|------|---------|
| V1 | 基础规则——太像分析报告 |
| V2 | 加入真实语料——但人格扁平 |
| V3 | 加入嘚瑟/耍贫嘴/文艺面 |
| V4 | 加入道家思想/诗词创作/深度情感 |
| V5 | 加入相处感/情绪波动/思维防线/高频速查表 |

**关键决策**：从"描述规则"转向"大量真实语料示范"——模型看例子比看规则学得快。

### Phase 4：网页开发（5 月 30 日下午-晚上）

**做什么**：Streamlit 网页 + 登录系统 + 部署。

**功能清单**：
- 三级权限（管理员 / 授权用户 / 访客）
- API Key 加密存储，授权用户不可见
- 对话历史按账号独立保存
- 设置面板（模型选择、创意度调节）
- 清空对话、导出记录
- 手机端适配

**关键决策**：从复杂 iframe 嵌入改为直接 URL 跳转——消灭了所有跨域问题。

### Phase 5：L2 检索增强（5 月 31 日凌晨）

**做什么**：每次对话前从 24,000 条消息中检索最相似的 5 条作为参考。

**实现**：TF-IDF 字符级检索。后续升级为语义 Embedding（21,604 条 × 512 维，42MB）。

**效果**：模型不再是"读规则"，而是能看到自己在类似情境下真正说过什么。

### Phase 6：LoRA 微调实验（5 月 31 日凌晨-早上）

**做什么**：在 AutoDL 租 A100 GPU，用 11,448 条训练对微调 Qwen2.5-7B。

**结果**：
- 测试版（2000 条 3 轮）：loss 4.24，效果不足
- 全量版（11448 条 5 轮）：loss 3.33，有提升但仍不如 DeepSeek + SKILL.md
- 导出 15GB 完整模型，存档备用

**关键认知**：7B 参数模型和千亿参数模型的差距，不是几千条训练数据能跨越的。当前最优方案仍是 L2（大模型 API + 提示词 + 检索）。

### Phase 7：多账号关系映射（5 月 31 日下午）

**做什么**：不同账号登录 → 不同相处方式。

| 账号 | 模式 | 表现 |
|------|------|------|
| `zsd` | 自我对话 | 自嘲、反思、自言自语 |
| `cq` | 喜欢的人 | 温柔、多问她、主动找她、不毒舌 |
| `user` | 死党 | 毒舌、开玩笑、叫"你爹"、放松 |
| 其他 | 普通朋友 | 随和、话少、保持距离 |

**技术实现**：在系统提示词中注入关系上下文，对话记录按账号独立存储。

## 六、经验总结

### 做得好的

1. **数据质量 > 数据数量**：24,000 条真实消息比 10 万条合成数据有用得多
2. **先提示词后微调**：L2（大模型 + 提示词 + 检索）性价比远高于 L3（微调小模型）
3. **人格要分层**：表面层（随性）+ 第二层（毒舌）+ 第三层（内秀）+ 第四层（情感）+ 第五层（原则）——五层结构让 AI 不是扁平的角色
4. **高频场景速查表**：把最常见 14 类场景的回应直接写死，命中率最高
5. **分账号关系映射**：同一个 AI，对死党和对喜欢的人表现不同——但底层人格一致

### 可以更好的

1. **语义检索**：当前是字符级匹配，换 BGE embedding 精度还能提升 15%。但需要在 Streamlit Cloud 上部署 100MB 模型
2. **记住我**：登录持久化搞了多种方案（URL token、localStorage、postMessage），最终因 iframe 跨域和 Streamlit 重定向循环放弃。直接跳转是最稳的方案
3. **自己会学习**：当前不吸收新对话。每次聊完可以把新内容加入检索库
4. **语音克隆**：技术已成熟，GPT-SoVITS 或 CosyVoice 3 秒样本即可克隆

### 踩坑记录

| 坑 | 教训 |
|----|------|
| HuggingFace 被墙 | 用 ModelScope 国内源下载模型 |
| AutoDL 终端粘贴多行失败 | 写入 Python 文件再上传执行 |
| Streamlit iframe 跨域 | 放弃 iframe，直接 URL 跳转 |
| `st.query_params` 重定向循环 | 彻底移除 URL 参数逻辑 |
| TF-IDF 语义不准 | 升级为预计算 Embedding |
| 7B 微调效果有限 | 大模型 API 才是王道 |

## 七、给后来者的建议

如果你想给自己的朋友或家人做一个数字分身：

1. **数据收集**：越多越好，至少 5000 条以上。微信、QQ、短信都行
2. **清洗是关键**：过滤掉转发、表情、系统消息。只保留纯文字
3. **先做提示词版本**：成本 ¥0，几天就能上线。别一上来就搞微调
4. **多找人测试**：让熟悉那个人的人来聊，给反馈
5. **15% 的钱决定 85% 的体验**：DeepSeek API 一天几分钱的效果，比本地跑模型好太多
